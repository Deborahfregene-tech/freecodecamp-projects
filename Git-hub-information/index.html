Introduction

Git is a distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

  • Version Control
  • Distributed Architecture
  • Integrity

GitHub focuses on storing your projects, such as an HTML or CSS based experiment.

The code will be stored in the exact way you have written it in your software e.g.

x = 5; y = 6; z = x + y
Configuring and Initializing

Before you start tracking files, you need to introduce yourself to Git so your commits are associated with your name.

Once configured, you can turn any local folder on your computer into a tracking Git repository.

  • Set your global username.
  • Set your global email address.
  • Initialize a blank repository inside your project's root folder.
git config --global user.name "Your Name"
The Staging Area and Committing

Git has a unique concept called the "Staging Area" (or index). It acts as a draft space where you gather and prepare your changes before permanently saving them.

Think of staging as organizing items into a box, and committing as sealing that box with a label.

  • Modified: You change files in your working directory.
  • Staged: You mark the modified files to go into your next commit.
  • Committed: Git permanently saves the snapshot of your staged files to the database.
git status git add filename.txt git commit -m "Add a descriptive commit message"
Working with Branches

Branching means you diverge from the main line of development and continue to do work without messing with that main line.

In Git, branches are incredibly lightweight and fast. They are simply pointers to specific commits.

  • Feature Isolation: Work on a new feature or bug fix in a safe sandbox.
  • Collaboration: Multiple developers can work on different parts of the code simultaneously.
git branch feature-login git checkout feature-login
Merging Changes

Once you have finished working on your feature branch and have thoroughly tested it, you need to bring those changes back into your main codebase. This process is called merging.

Git will look at the history of both branches and combine the code automatically.

    Merge Types:
  • Fast-Forward Merge: Occurs if the main branch hasn't had any new commits since you branched off. Git just moves the pointer forward.
  • Three-Way Merge: Occurs when both branches have diverged with new commits. Git creates a new "merge commit" to tie them together.
git checkout main git merge feature-login
